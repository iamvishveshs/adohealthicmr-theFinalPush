# 413 Request Entity Too Large - Fix

## Problem
Getting `413 Request Entity Too Large` error when uploading 252MB video file.

## Root Cause
Next.js App Router route handlers have a default body size limit. When `request.formData()` is called, Next.js tries to parse the entire request body into memory, which hits this limit for large files.

## Solution Applied

### 1. ✅ Simplified Upload Flow
- Removed unnecessary file processing
- Pass File object directly to Cloudinary
- Let Cloudinary handle streaming internally

### 2. ✅ Increased Node.js Memory
Updated `package.json` to increase Node.js memory:
```json
"dev": "NODE_OPTIONS='--max-old-space-size=4096' next dev"
```

### 3. ✅ Direct Cloudinary Upload from Server
- Server receives FormData from browser
- Server immediately forwards File object to Cloudinary
- No intermediate processing or memory loading

## How It Works Now

```
Browser → Next.js (receives FormData) → Cloudinary (streams file)
```

The File object is passed through without loading into memory.

## Testing

1. **Restart your dev server:**
   ```bash
   npm run dev
   ```

2. **Try uploading your 252MB video again**

3. **If still getting 413:**
   - The issue might be that `request.formData()` still loads the body
   - Next.js App Router has a hard limit
   - **Solution:** Compress the video before uploading (recommended)

## Alternative Solutions

### Option 1: Compress Video (Recommended)
- Use HandBrake to compress to ~50-100MB
- Much faster uploads
- Better user experience

### Option 2: Use Direct Browser Upload
- Fix CORS on Cloudinary side
- Upload directly from browser (bypasses Next.js)

### Option 3: Custom Server
- Use a custom Express server
- Configure body-parser with higher limits
- More complex setup
